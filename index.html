Linux time holds 8 hours differnce with Windows in dual system

Posted on 2018-08-07 | In Problem and Solution

What we find

  1. After synchoronizing Windows local time with online time, we find the Linux time is 8 hours later then the real time.
  2. After synchoronizing Linux local time with online time, we find the Windows time is 8 hours earier then the real time.

Why it happens

  1. Windows time = Hardware time
  2. Linux time=UTC(Coordinated Universal Time)+UTC offset，UTC = Hardware time。

What we need to do

We advise to let Windows time = UTC + UTC offset：

  1. Run cmd.exe as adminstrator, its loaction is C:\Windows\System32\cmd.exe。
  2. Enter the following command and execute：

Reg add HKLM\SYSTEM\CurrentControlSet\Control\TimeZoneInformation /v RealTimeIsUniversal /t REG_DWORD /d 1

Linux向用户组添加已登录的用户失败

Posted on 2018-08-06 | In Problem and Solution

发现过程

nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ sudo addgroup test
正在添加组"test" (GID 1003)...
完成。
nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ sudo gpasswd -a nanjiang test
正在将用户“nanjiang”加入到“test”组中

以上命令行的目的是创建新的用户组test并将，登陆的用户nanjiang按道理应该在test用户组中，然而我们发现当前用户并不在test用户组中。

nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ id
uid=1000(nanjiang) gid=1000(nanjiang) 组=1000(nanjiang),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),116(lpadmin),126(sambashare),1002(shared)

然而，当我们用su命令再登陆nanjiang用户时，我们发现用户在test用户组中。

nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ su nanjiang
密码： 
nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ id
uid=1000(nanjiang) gid=1000(nanjiang) 组=1000(nanjiang),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),116(lpadmin),126(sambashare),1002(shared),1003(test)

这究竟是怎么回事呢？

原因分析

我们可以猜测，Linux会在用户登陆的时候，加载登陆用户的组信息并保存下来。之后调用id命令时，会使用缓存的数据。所以su命令重登陆后，nanjiang多了一个所属的用户组test。

解决方案

重新登陆当前用户，用户新的组信息就会保存下来。

nanjiang@nanjiang-RESCUER-Y720-15IKB:~$ id
uid=1000(nanjiang) gid=1000(nanjiang) 组=1000(nanjiang),4(adm),24(cdrom),27(sudo),30(dip),46(plugdev),116(lpadmin),126(sambashare),1002(shared),1003(test)
